# ERC721

- ERC721 借鉴了ERC20标准经验的教训
- 标准里引入`safeTransferFrom`，加入对接收者 `onERC721Received()`函数调用
  - 通知接收者执行相应代码，同时避免NFT锁在合约地址里
- 加入了接口提测（ERC165），在合约中暴露自己实现了哪些方法
  - `function supportsInterface(bytes4 interfaceID) external view returns (bool)`
- IPFS: (取代HTTP) 去中心存储协议，按内容寻址
  - FileCoin: 去中心化存储区块链
  - IPFS 浏览器插件（ipfs:xxx 自动跳转）
- Arweave: 去中心化存储区块链，每个节点都是一个http服务器

## ERC777 Openzeppelin 最新的已经去掉了

- `send(to, value, data)`
- 防止误入合约被锁死
- 即便是普通地址也可以实现回调
- ERC777 通过全局注册表合约 (ERC1820) 的注册监听回调

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;
function send(from, to, amount,...) {
  _callTokensToSend(operator, from, to, amount, ...)
}
// 检查是否有注册回调，有的话就调用一下
function _callTokensReceived(operator, from, to, amount) private {
  address implementer = _ERC1820_REGISTRY.getInterfaceImplementer(to, _TOKENS_SENDER_INTERFACE_HASH);
  if(implementer != address(0)) {
    IERC777Sender(implementer).tokensToSend(operator, from, to, amount, ...);
  }
}
```

## ERC1363

- Payable Token
- 3对额外函数
  - transferAndCall
  - transferFromAndCall
  - approveAndCall
 
    https://github.com/vittominacori/erc1363-payable-token
    https://learnblockchain.cn/article/8549

```solidity
function transferWithCallback(address to, uint256 amount) external returns (bool) {
  _transfer(msg.sender, to, amount);
  if(to.isContract()) {
    bool rv = TokenRecipient(to).tokensReceived(msg.sender, amount);
    require(rv, "No tokensReceived");
  }
  return true;
}
```
