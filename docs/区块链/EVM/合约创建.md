# 合约创建

- 创建的合约地址是如何生成的
- 如何部署到一个确定的地址上
- 大规模部署相同功能的合约如何节省Gas
  - EIP-1167 最小代理合约
  - EIP-2535 钻石代理

https://www.evm.codes/

## 地址计算

- CREATE
  - 根据创建者(sender)的地址以及创建者发送过的交易数量(nonce)来计算确定
    - `keccak256(rlp.encode([normalize_address(sender), nonce]))[12:]`
    - 不易控制合约地址，多链要保持一致的合约地址麻烦， 适合于不需要预测的情况
- CREATE2
  - 只能用合约创建
  - 根据创建者（工厂合约地址）的地址 + 用户设置的salt及要创建的合约字节码确定
    - `keccak256(0xff + sender + salt + keccak256(init_code))[12:]`

```solidity
initialisation_cdoe = memory[offset:offset+size]
address = keccak256(0xff+sender_address + salt + keccak256(initialisation_code))[12:]

function create2Contract(uint _salt) public returns (address) {
  C c = new C{salt: keccak256(abi.encode(_salt))}();
  return address(c);
}
```

- CREATE3 库
  - CREATE2 有一个致命缺陷：创建的地址与合约字节码强绑定，实际开发中，在不同链部署时，构造函数的参数可能不一样，这样的话字节码也不一样了
  - 创建 不与特定合约字节码绑定的确定性合约地址
    - 场景：更容易实现多链地址一致性，只和salt有关
  - 原理：工厂先用create2部署一个部署器，在用部署器create方式部署用户合约
    - `https://github.com/transmissions11/solmate/blob/main/src/utils/CREATE3.sol`

```solidity
import {CREATE3 } from "https://github.com/transmissions11/solmate/blob/main/src/utils/CREATE3.sol";

contract C {

}


function create3Contract(uint _salt, bytes memory creationCode) external returns (address deployed) {
  deployed = CREATE3.deploy(
    keccak256(abi.encode(_salt)),
    creationCode,
    0
  );
}
```

- 最小代理 创建合约
  - 多个代理委托给同一个实现合约
  - EIP-1167 提案
  - https://github.com/optionality/clone-factory

```solidity
function createClone(address prototype) internal returns (address proxy) {
  bytes20 targetBytes = bytes20(prototype);
  assembly {
    // 0x40 是 Solidity 的“自由内存指针”，指向可用内存起始位置
    let clone := mload(0x40)
    // 3d602d80600a3d3981f3
    // 对应：
    // [PUSH1 0x2d] [DUP1] [PUSH1 0x0a] [PUSH1 0x3d] [CODECOPY] [RETURNDATASIZE] [DUP1] [PUSH1 0x3d] [RETURNDATASIZE] [CODECOPY] [RETURNDATASIZE] [RETURN]
    // 从当前合约的代码中，复制一段代码到内存并返回
    mstore(clone, 0x3d602d80600a3d3981f3363d3d373d3d3d363d73000000000000000000000000)
    // 在偏移量 0x14 处写入 20 字节的目标合约地址
    mstore(add(clone, 0x14), targetBytes)
    // 固定后缀
    mstore(
        add(clone, 0x28),
        0x5af43d82803e903d91602b57fd5bf30000000000000000000000000000000000
    )
    // create(0, clone, 0x37)是 EVM 的 CREATE操作码
    proxy := create(0, clone, 0x37)
  }
  return proxy;
}
```
